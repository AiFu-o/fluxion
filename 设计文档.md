# Fluxion 设计文档

Fluxion是一个用于构建用户界面的 JavaScript 框架，设计目标：

* **AI友好**：生成页面使用的字符 token 少，易理解
* **人类友好**：可维护，阅读直观
* **高性能**：响应式 + 编译优化

---

## 一、包结构

```
fluxion
├─ packages
│  ├─ fluxion                # 主入口（用户 npm install 的包）
│  ├─ runtime-core           # 平台无关 runtime
│  ├─ runtime-dom            # DOM renderer
│  ├─ reactivity             # 响应式系统（参考 Vue）
│  ├─ compiler-core          # DSL 编译核心
│  ├─ compiler-dom           # DOM 相关编译
│  ├─ compiler-nui           # .nui 文件编译器
│  ├─ shared                 # 公共工具
│  └─ vite-plugin-fluxion    # Vite 插件
│
├─ playground
│  └─ sfc                    # SFC playground（.nui 实验环境）
├─ docs                      # 框架文档
├─ memory                    # 设计记录 / 计划
└─ build                     # 构建脚本

```

## 三、nui文件编译流程
```text
            .nui file
                │
                ▼
        compiler-nui
        (DSL parser)
                │
                ▼
               AST
                │
                ▼
        compiler-core
        (AST transform)
                │
                ▼
        compiler-dom
       (DOM transform)
                │
                ▼
            codegen
                │
                ▼
        render function
```

## 四、Runtime 执行流程
生成的 `render function` 会在 runtime 执行：


```text
        render function
               │
               ▼
          runtime-core
        (component system)
               │
               ▼
             vnode
               │
               ▼
            renderer
               │
               ▼
          runtime-dom
               │
               ▼
              DOM
```

## 五、响应式数据流

当 state 改变：

```ts
count++
```

触发：

```text
        state change
             │
             ▼
         reactivity
        (signal/effect)
             │
             ▼
       component update
             │
             ▼
           render()
             │
             ▼
           vnode diff
             │
             ▼
          runtime-dom
             │
             ▼
           DOM patch
```

## 六、语法规范

### 1. 缩进与语法

* **缩进**：使用 Tab 缩进
* **属性**：`name="test"` 带引号
* **view.children**：解析成 AST 时为文本数组

### 2. 响应式声明

删掉 `state` 块，signal/asyncSignal 可直接声明：

```nui
count = signal(0)
users = asyncSignal(fetchUsers)
name = signal("test")
```

### 3. 类型推断

* 父组件传入 signal → 类型为 `Signal<T>`
* 传入字符串/数字 → 自动推断
* 也可显式声明类型

### 4. 事件处理

使用 `@事件名` 语法：

```nui
button @click=increment
```

编译为：`onClick: increment`

### 5. emit 函数

组件可定义 emit：

```nui
emit("update", payload)
```

### 6. 控制流

| DSL 写法 | JS 编译 |
| ------- | ------- |
| `if expr` | `expr ? ...` |
| `elif expr` | `expr ? ...`（放在 else 里） |
| `else` | `: ...` |
| `for item in list` | `list.map(item => ...)` |

条件 + 循环可嵌套

## 九、编译输出

### 1. Signal 初始化

```ts
import Title from "./Title.nui"
import { signal, asyncSignal, h, emit } from "fluxion-runtime"

const count = signal(0)
const name = signal("test")
const loading = signal(true)
```

### 2. 函数逻辑

```ts
function increment() {
	count.update(c => c + 1)
}

function handleUpdate(v) {
	emit("update", v)
}
```

### 3. render function

```ts
function render() {
	return h("div", [
		h("p", "hello world"),
		h("p", ["Count ", count()]),
		loading()
			? h("p", "loading...")
			: h("p", "loaded"),
		users().map(user => h("p", [user.name])),
		h(Title, { name: name() }, ["hello"]),
		h("button", { onClick: increment })
	])
}
```

> ⚠️ 注意：
>
> * `{count}` 被转换为 `count()`，读取 signal 的值
> * `if/else` 编译为三元表达式
> * `for item in list` 编译为 `list.map(item => ...)`

## 十、打包策略

### 构建工具

使用 **Rollup** 打包，合理使用 `/*#__PURE__*/` 标记

### 输出格式

* **npm / Vite / Rollup**：ESM + 多入口 + tree-shaking
* **CDN**：fluxion.global.js → 单一 bundle（IIFE）

### 类型输出

支持 TypeScript 类型输出（.d.ts）

## 十一、要求

* 错误处理：每个函数需要完整的 Warn 提示和错误捕获
* 测试：每个模块都需要完整的单元测试，在撰写单元测试时请分按功能拆分测试文件，禁止将一个功能的所有测试放在一个文件中

* 支持 effect
